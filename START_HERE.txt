============================================================
  WEBSITE — charliehane.com
============================================================

This folder IS your website. Every page, every piece of art,
every line of text. Opening any HTML file in a browser shows
that page; pushing the whole folder to Cloudflare deploys it.

============================================================
  WHAT YOU EDIT (the only things you should ever touch)
============================================================

  Editable Text Content.json           All text on the site — your bio,
                         film titles, descriptions, contact
                         email, quotes, menu labels. Edit
                         the JSON, save, refresh browser.

  assets/                The 14 SVG art files. Replace any
                         file with a new SVG of the same name
                         and the site picks it up automatically.
                         See Assets_Specs/ for what each one is
                         and what's required.

  lotties/               After Effects animations exported via
                         Bodymovin. Drop a .json file here with
                         a name listed in Assets_Specs/*.lottie.todo
                         and it replaces the matching CSS / JS
                         animation on the site. Start with
                         wind-streak.json (the only one wired
                         up so far).

============================================================
  WHAT YOU REFERENCE (read-only — your private notes)
============================================================

  Assets_Specs/          Spec stubs for everything swappable.
                         Two flavors:
                           *.svg.todo     — for static art in
                                            assets/. viewBox,
                                            required class
                                            names, design notes.
                           *.lottie.todo  — for AE animations in
                                            lotties/. Comp size,
                                            frame rate, loop
                                            behavior, Bodymovin
                                            export settings.
                         Open the stub BEFORE you start drawing
                         or animating. The _CHECKLIST.txt is
                         your overview.

============================================================
  WHAT YOU DON'T TOUCH (the engine — let Claude handle these)
============================================================

  building-blocks/css/   All visual styling. Colors, layouts,
                         fonts, animations. Live behind every
                         page.

  building-blocks/js/    All interactivity. The bike riding,
                         brick exploding, cursor spinning,
                         skeleton eyes glowing, charlie kicking,
                         hell flames flickering — all of it.

  *.html                 The four page entry points. They tie
                         everything together. Structural edits
                         (adding a brand new section, etc.)
                         live here, but day-to-day you ignore
                         them.

============================================================
  ABOUT THE ANIMATIONS (read this once, it'll save confusion)
============================================================

  There are NO gif files anywhere in this website. There never
  were. Every moving thing on the site is generated by CODE in
  real time, not played back from an animated image.

  This matters because it changes what "swap a piece of art"
  means: you replace the still drawing, the code keeps doing
  the moving. The clothes, not the choreography.

  WHAT'S DRIVING EACH MOVING THING:

    Continuous loops (clouds drifting, flames flickering,
    cursor spinning, brick rubble falling, bike jumping)
       ↓
    These are CSS animations. Defined in
    building-blocks/css/style.css.

    Interactive motion (bike crash, brick explosion,
    skeleton eyes glowing on hover, charlie kicking,
    dig tunnels appearing as you mouse around)
       ↓
    These are JavaScript animations. Defined in
    building-blocks/js/*.js. They have to react to your
    scroll position / clicks / hovers, so they can't be
    pre-rendered.

  WHAT YOU CAN SWAP:

  ▸ The still drawing (the "frame") of any animated piece.
    Example: redraw `bike-with-rider.svg` and the wheels in
    your new drawing still rotate, because the code targets
    anything labeled `class="bike-wheel"`. The required
    class names are listed in each Assets_Specs/ stub.

  WHAT YOU CAN'T SWAP (without asking Claude):

  ▸ HOW something moves. E.g. "make the bike do a backflip
    instead of crashing" → that's a code change in
    building-blocks/js/preproduction.js, not an art swap.

  ▸ The TIMING / SPEED of an animation. E.g. "make the
    wheels spin faster" → CSS change in style.css.

  ▸ Replacing animation with a GIF or video. The current
    setup doesn't load .gif or .mp4 files. Possible to add,
    but it's a structural change.

  IF A DESIGNER GIVES YOU AN "ANIMATED SVG":

  Some designers can deliver SVG files that already have
  motion built into them (using <animate> tags inside the
  SVG itself). These work fine for the always-looping
  decorative pieces — clouds, flames, drifting birds —
  but NOT for interactive pieces (the bike crash, the
  brick explosion, etc.) because those need to listen to
  the user.

  If you ever get one of those: just drop it into assets/
  with the matching filename. It plays its own motion
  automatically. (If it conflicts with the existing CSS/JS
  motion, ask Claude to silence one or the other.)

============================================================
  HOW TO PREVIEW LOCALLY
============================================================

  Open Terminal, type:

    cd ~/Desktop/WEBSITE
    python3 -m http.server 5173

  Then visit http://localhost:5173/ in your browser.

  Leave the terminal window open while you work. Hard-refresh
  the browser (Cmd+Shift+R) after each edit to see changes.

  When you're done, Ctrl+C in the terminal to stop the server.

============================================================
  HOW TO DEPLOY TO charliehane.com
============================================================

  One-time setup (do this once):
    1. Make a GitHub account at github.com
    2. Push this folder to a new GitHub repo
    3. Connect it to Cloudflare Pages (pages.cloudflare.com)
    4. Point charliehane.com's DNS at Cloudflare

  After that, the workflow is:
    1. Edit a file locally
    2. In Terminal:
         cd ~/Desktop/WEBSITE
         git add .
         git commit -m "what you changed"
         git push
    3. Cloudflare auto-deploys in ~30 seconds
    4. Refresh charliehane.com

  (If any of those steps look scary, ask Claude to walk you
  through them the first time. After that you'll have muscle
  memory.)

============================================================
  QUICK REFERENCE — common tasks
============================================================

  "I want to add a new film to Preproduction"
    → Edit Editable Text Content.json, find "preproduction" → "films",
      add a new entry to the array with scene, sceneNum,
      tag, title, meta. Save. Refresh browser.

  "I drew a new bike — how do I use it?"
    → Save your file as `bike-with-rider.svg` and replace
      assets/bike-with-rider.svg. Refresh browser.

  "I want to change my contact email"
    → Editable Text Content.json → site → email. Save. Refresh.

  "I want to reorder the menu at the bottom of a page"
    → Editable Text Content.json → menus → afterPreproduction (or
      whichever menu). Reorder the array entries.

  "I made a Lottie in After Effects, how do I use it?"
    → Export via Bodymovin → save the .json into the lotties/
      folder with the exact filename listed in the matching
      Assets_Specs/<name>.lottie.todo file. Hard-refresh.

  "I want a new ANIMATION slot wired up for Lottie swap"
    → Ask Claude. (Currently only wind-streak is wired up;
      adding more is straightforward but needs code changes.)

  "I want a brand new section / page / animation"
    → Ask Claude. That's a structural change.
